Mythic GME 2E Drama Hack

The following changes made to the official rules for focus threads and chaos factor are meant to encourage a more self-contained story that feels akin to a single novel, movie, or episode. This method can be used in succession to create a series.


Focus Threads

  • More than one focus thread is allowed—the primary overarching thread is referred to as the Main Plot thread and the rest are Subplot threads. Each is assigned a label along with their type, such as "Subplot: Find a Mentor."
  • Retroactively creating a focus thread from a reoccurring theme is also allowed. On creation, fill in boxes on the progress track according to events that have already transpired, as if it had been tracked from the start.
  • Subplot threads are nested. Subplot A and Subplot B are contained within the Main Plot, and Subplot A1 and Subplot A2 are contained within Subplot A, etc. A parent thread cannot be resolved until all of its child threads are resolved. Therefore, only create threads that are intended to be resolved before the conclusion of the Main Plot.
  • For clarity of intent, threads have three properties that should be defined: Premise (the question or event that is expected to be resolved at the thread's conclusion), Goal (how the character intends to resolve it), and Problem (what is complicating their attempt).
  • Additionally, the premise of each thread has an element type that helps specify which kind of notable events move the thread toward its conclusion, whether they be positive or negative: Milieu (driven by place), Inquiry (driven by answers), Character (driven by angst), and Event (driven by action). Ideally, each thread should be a different element.
  • Any scene that moves the thread closer to its conclusion, whether positively or negatively, counts as progress. Flashpoints are triggered only when an action is taken that will result in recording enough progress points to complete a phase (a 5-point block), regardless of whether they are recorded immediately or during the end of scene bookkeeping. Nothing else counts as a flashpoint.
  • Flashpoint random events are always a dramatically negative development for the focus thread, and they must be generated before the end of the scene that triggered the flashpoint. The flashpoint random event may use the Current Context event focus when generated immediately following the action that triggered it, or it may roll on the event focus table when generated later in the scene (usually near the end, leading into the next scene).


Chaos Factor

The chaos factor is set according to progress on the Main Plot focus thread track. CF starts at 5, then becomes 6 and 7 once it fills those boxes. CF resets to 5 again once the Main Plot thread is concluded. Subplot threads do not affect the chaos factor.

Fate Questions

Questions should be phrased so that a Yes answer is the most dramatic result, whether that is positive or negative for the PC. This way the drama increases toward the climax of the Main Plot focus thread, and then goes away during its resolution, or denouement.
